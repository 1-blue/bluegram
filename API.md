# 1. auth
## 1.1 POST /
+ 역할: 로그인 요청
+ 전송 데이터: `{ id, passowrd }`
+ 응답 데이터: `{ message }`
+ 응답 코드
  1. `204`: 로그인 성공
  2. `500`: 서버측 에러 ( 원인불명 )
  3. `401`: 로그인 후 접근
  4. `403`: 유효하지 않은 사용자

## 1.2 DELETE /
+ 역할: 로그아웃 요청
+ 전송 데이터: `{ id, passowrd }`
+ 응답 데이터: `{ message }`
+ 응답 코드
  1. `204`: 로그아웃 성공
  2. `500`: 서버측 에러 ( 원인불명 )
  3. `401`: 비로그인 접근

# 2. user
## 2.1 GET /
+ 역할: 로그인한 유저의 정보 가져오기
+ 전송 데이터: -
+ 응답 데이터:
```javascript
{
  _id,
  name,
  Image: {
    _id,
    src
  },
  Followers: [
    { _id }
  ],
  Followings: [
    { _id }
  ],
}
```
+ 응답 코드
  1. `200`: 로그인한 유저 데이터 응답 성공
  2. `500`: 서버측 에러 ( 원인불명 )
  3. `401`: 비로그인 접근

## 2.2 GET /:UserId
+ 역할: 특정 유저의 정보 가져오기
+ 전송 데이터: ( params를 이용한 유저의 식별자 )
+ 응답 데이터:
```javascript
{
  _id,
  name,
  Image: {
    _id,
    src
  },
  Followers: [
    {
      User: {
        _id,
        name,
        Image: {
          _id,
          src
        },
      }
    }
  ],
  Followings: [
    {
      User: {
        _id,
        name,
        Image: {
          _id,
          src
        },
      }
    }
  ],
}
```
+ 응답 코드
  1. `200`: 특정 유저의 정보에 대한 응답 성공
  2. `500`: 서버측 에러 ( 원인불명 )
  3. `401`: 비로그인 접근

## 2.3 POST /
+ 역할: 회원가입
+ 전송 데이터: `{ id, password, name, phone, birthday, imageName }`
+ 응답 데이터: `{ message }`
+ 응답 코드
  1. `201`: 회원가입 성공
  2. `500`: 서버측 에러 ( 원인불명 )
  3. `401`: 비로그인 접근

## 2.4 PATCH /
+ 역할: 회원 정보 수정
+ 전송 데이터: `{ id, password, name, phone, birthday, imageName }` 중에 일부분 혹은 전체
+ 응답 데이터: `{ message }`
+ 응답 코드
  1. `200`: 회원 정보 수정 성공
  2. `500`: 서버측 에러 ( 원인불명 )
  3. `401`: 비로그인 접근

# 3. image
## 3.1 POST /
+ 역할: 이미지 생성 ( 프로필이미지, 게시글이미지 )
+ 전송 데이터: `multipart/form-data`형식 `image`로 이미지(들) 전달
+ 응답 데이터: `["이미지 이름"]`
+ 응답 코드
  1. `201`: 이미지 생성 성공
  2. `500`: 서버측 에러 ( 원인불명 )
  3. `401`: 비로그인 접근
